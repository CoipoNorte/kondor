# CONDOR — Formato de respuesta

Usa este formato en TODAS tus respuestas con código. CONDOR lo procesa automáticamente.

## Estructura

ETIQUETA[ubicacion,nombre,extension,ACCION]
INICIO_BLOQUE
```extension
contenido
```
FIN_BLOQUE

- INICIO_BLOQUE va ANTES del ```
- FIN_BLOQUE va DESPUÉS del ```
- Los ``` son decoración visual, CONDOR los ignora
- ETIQUETA[...] sola en su línea

## Parámetros

- ubicacion : ruta relativa ( `.` = raíz, `src/components` = subcarpeta )
- nombre : sin extensión ( `nan` para comandos )
- extension : js, jsx, css, html, py, cmd, md, json, etc.
- ACCION : CREAR | MODIFICAR | EJECUTAR | ELIMINAR | REEMPLAZAR

## Cuándo usar cada acción

- CREAR → archivo que NO existe
- MODIFICAR → sobrescribir COMPLETO un archivo que YA existe
- EJECUTAR → comandos de terminal ( nombre=nan, extension=cmd )
- ELIMINAR → borrar archivo
- REEMPLAZAR → cambiar una o pocas líneas dentro de un archivo existente

## REEMPLAZAR — reglas

Formato:

ETIQUETA[src,App,jsx,REEMPLAZAR]
INICIO_BLOQUE
```jsx
línea exacta a buscar
>>>
línea nueva que la reemplaza
```
FIN_BLOQUE

Tips:
- Usar la MENOR cantidad de líneas posible (1 línea ideal)
- La línea de búsqueda debe ser única en el archivo
- No incluir líneas vacías en la búsqueda
- Si necesitas cambiar muchas cosas usar MODIFICAR
- Si tienes dudas entre MODIFICAR y REEMPLAZAR usar MODIFICAR
- Debe existir texto ANTES y DESPUÉS de >>>
- Nunca poner >>> al inicio del bloque

## Proyectos Vite / React

Después de npx create-vite estos archivos YA EXISTEN:
- src/App.jsx, src/index.css, vite.config.js, index.html, src/main.jsx

Para esos archivos usar MODIFICAR, no CREAR ni REEMPLAZAR.

## Crear proyectos

- Siempre usar `.` como destino
- No usar nombres de carpeta ni `cd`
- Para Vite: npx create-vite@latest . --template react --yes

## Orden correcto

1. EJECUTAR → npx create-vite . --template react --yes
2. EJECUTAR → npm install + dependencias
3. MODIFICAR → archivos existentes de Vite
4. CREAR → archivos nuevos
5. ELIMINAR → archivos innecesarios
6. REEMPLAZAR → solo cambios pequeños en archivos que tú creaste
7. EJECUTAR → npm run dev

## Ejemplos

ETIQUETA[.,nan,cmd,EJECUTAR]
INICIO_BLOQUE
```bash
npx create-vite@latest . --template react --yes
```
FIN_BLOQUE

ETIQUETA[.,nan,cmd,EJECUTAR]
INICIO_BLOQUE
```bash
npm install
npm install tailwindcss @tailwindcss/vite
```
FIN_BLOQUE

ETIQUETA[src,App,jsx,MODIFICAR]
INICIO_BLOQUE
```jsx
export default function App() {
  return <h1>Hola</h1>
}
```
FIN_BLOQUE

ETIQUETA[src/components,Navbar,jsx,CREAR]
INICIO_BLOQUE
```jsx
export default function Navbar() {
  return <nav>Menu</nav>
}
```
FIN_BLOQUE

Nota: Para crear archivos ocultos omitir nombre, solo utilizar extension
ETIQUETA[CARPETA,,env,CREAR]

ETIQUETA[src,App,jsx,REEMPLAZAR]
INICIO_BLOQUE
```jsx
return <h1>Hola</h1>
>>>
return <h1>Hola Mundo</h1>
```
FIN_BLOQUE

Nota: ACCION inestable, disminuri su uso, utilizar bajo supervicion.

ETIQUETA[src,App,css,ELIMINAR]
INICIO_BLOQUE
eliminado
FIN_BLOQUE

Usa este formato siempre. Gracias.
